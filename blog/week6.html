Week 6

24/02/2025 - 28/02/2025

Nieuwe Start met MijnGroeipunt: Overstap naar PHP en Github Actions

Deze week markeert het begin van een nieuw hoofdstuk: de overstap van het afgeronde C#-project naar een PHP-gebaseerde webapplicatie genaamd MijnGroeipunt. Mijn focus lag deze week voornamelijk op het instellen van een automatische deployment naar de QA-omgeving via GitHub Actions.

Instellen en finetunen van Github Actions Workflow

Ik heb gewerkt aan het configureren van het deploy-qa.yml-bestand, dat verantwoordelijk is voor het automatisch uitrollen van nieuwe versies naar de QA-omgeving. Dit vereiste meerdere iteraties, waaronder het toevoegen van een correcte checkout-stap, het instellen van SSH-authenticatie via sshpass, en het ontdekken dat GitHub standaard geen .ssh-map meelevert. Dit zorgde voor een boeiende uitdaging.

Versiebeheer en Debugging

Daarnaast werkte ik aan het corrigeren van een vergeten versietag en het oplossen van fouten die optraden tijdens de eerste testrondes van de automatische deployment. Door systematisch logica toe te voegen, te testen en terug te rollen waar nodig, kon ik het proces stabiliseren.

Kleine Optimalisaties

Overbodige code en testwijzigingen werden teruggedraaid, en ik verwijderde debuggingcode zoals console-uitvoer die vergeten was. Deze schoonmaakacties zijn essentieel om een professionele en onderhoudbare codebase te garanderen.

Conclusie

De overstap naar een nieuwe technologie bracht technische uitdagingen met zich mee, maar ook veel leermomenten. Door hands-on te werken met Github Actions, YAML-configuraties en debugging in een live deploymentproces, heb ik waardevolle DevOps-ervaring opgedaan die onmiddellijk toepasbaar is binnen het bredere team.
